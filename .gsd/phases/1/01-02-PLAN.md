  <action>
    Create internal/config/load.go in package config.

    Implement Load(path string) (*Snapshot, error):
    1. Open the file at path; wrap errors with fmt.Errorf("open config: %w", err).
    2. Decode with yaml.NewDecoder(f, yaml.DisallowUnknownField()).Decode(&cfg); wrap errors.
    3. Call validate(cfg) and return.

    Implement validate(cfg Config) (*Snapshot, error) with these checks in order:

    CHECK 1 — group_jid: Call types.ParseJID(cfg.Scope.GroupJID). Return error if parse fails. Also check jid.Server == types.GroupServer ("g.us"); if not, return error: "group_jid must be a group JID (got server %q)".

    CHECK 2 — owner_jids: For each entry in cfg.Scope.OwnerJIDs, call types.ParseJID. Return error on first failure, wrapping with the index and value.

    CHECK 3 — timezone: If cfg.Limits.QuietHours.Timezone is non-empty, call time.LoadLocation(cfg.Limits.QuietHours.Timezone). Return error on failure. Never use time.Local.

    CHECK 4 — cluster resolution: Build a map[string][]string from cfg.AnswersClusters (Name → Answers). For each matcher, look up Cluster key in the map. Return error if missing or if the cluster name appears more than once in answers_cluster (ambiguous reference).

    CHECK 5 — distance min-length: For each matcher word, enforce: distance 1 → word must be ≥5 runes (len([]rune(word)) ≥ 5); distance 2 → word must be ≥8 runes. Return error if any word violates the constraint, naming the matcher and word.

    CHECK 6 — self-loop guard: For each resolved matcher, for each answer string in its cluster, tokenize the answer on whitespace (strings.Fields), lowercase each token (strings.ToLower), and check if any token exactly matches (after lowercasing) any word in matcher.Words (also lowercased). Return error if a match is found: "answer in cluster %q contains keyword %q from matcher %q (self-loop)".

    Build the resolved Snapshot: populate Snapshot.Scope, Snapshot.Limits, Snapshot.Log, Snapshot.DB from cfg fields. Populate Snapshot.Matchers as []ResolvedMatcher — each ResolvedMatcher has Name, Words, Distance from MatcherConfig and Answers from the resolved cluster. Return the pointer.

    Imports: fmt, os, strings, time, github.com/goccy/go-yaml, go.mau.fi/whatsmeow/types, github.com/taldoflemis/bot-camomila/internal/config (same package — no import needed).
  </action>

    FILE 1 — internal/config/store.go (package config):

    Define type Store struct containing a single field: ptr atomic.Pointer[Snapshot].

    Implement:
    - NewStore(initial *Snapshot) *Store: creates Store, calls s.ptr.Store(initial), returns &s.
    - Get() *Snapshot: returns s.ptr.Load(). Add doc comment: "Get returns the current snapshot. Callers MUST hold the returned pointer for the full duration of one message-handling call."
    - Swap(n *Snapshot): calls s.ptr.Store(n). Add doc comment: "Swap atomically replaces the snapshot. Only the watcher goroutine calls this."

    Imports: sync/atomic only.

    FILE 2 — internal/config/watcher.go (package config):

    Define type Watcher struct with fields: store *Store, configPath string.

    Implement NewWatcher(store *Store, configPath string) *Watcher.

    Implement Run(ctx context.Context) error:
    1. Create fsnotify.NewWatcher(). If it fails, log WARN "fsnotify unavailable; using mtime poll" and jump to poll-only loop.
    2. Defer fw.Close().
    3. Compute dir := filepath.Dir(w.configPath) and base := filepath.Base(w.configPath). Call fw.Add(dir) — NEVER fw.Add(configPath) directly (atomic-rename editors break file-level watches).
    4. Record initial lastMtime via os.Stat(w.configPath).ModTime().
    5. Create poll := time.NewTicker(30 * time.Second). Defer poll.Stop().
    6. Declare var debounce *time.Timer.
    7. Main select loop (label this "mainLoop"):
       - case ctx.Done(): return nil
       - case ev, ok := <-fw.Events: if !ok, log WARN "fsnotify events channel closed; switching to mtime poll" and break to pollOnly loop. Filter: if filepath.Base(ev.Name) != base, continue. If not ev.Has(fsnotify.Write) && not ev.Has(fsnotify.Create) && not ev.Has(fsnotify.Rename), continue. If debounce != nil, debounce.Stop(). Set debounce = time.AfterFunc(200*time.Millisecond, w.reload).
       - case fwErr, ok := <-fw.Errors: if !ok || fwErr != nil, log WARN "fsnotify error; switching to mtime poll", "err", fwErr, and break to pollOnly loop.
       - case <-poll.C: call os.Stat(w.configPath). If ModTime().After(lastMtime), update lastMtime and call w.reload().

    pollOnly loop: for { select { case <-ctx.Done(): return nil; case <-poll.C: stat + mtime check + reload } }

    Implement reload() method:
    1. Call snap, err := Load(w.configPath).
    2. On error: slog.Warn("config reload failed; keeping previous config", "err", err). Return.
    3. On success: w.store.Swap(snap). slog.Info("config reloaded", "path", w.configPath).

    Imports: context, log/slog, os, path/filepath, time, github.com/fsnotify/fsnotify.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → config loader | Operator-supplied YAML file is read and parsed; malformed or adversarial YAML could cause panics or inject unexpected values |
| YAML values → JID / TZ parsing | Unvalidated strings passed to external parsers (types.ParseJID, time.LoadLocation) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | config.example.yaml / YAML load | mitigate | yaml.DisallowUnknownField() rejects unknown keys; goccy/go-yaml is a well-maintained strict decoder; validate() runs immediately after decode before any Snapshot is stored |
| T-02-02 | Elevation of Privilege | group_jid / owner_jids validation | mitigate | types.ParseJID validates JID format; group_jid additionally checked against types.GroupServer to prevent non-group JID from being accepted; owner JIDs validated format at load time |
| T-02-03 | Denial of Service | config reload loop | mitigate | 200ms debounce prevents reload storm on rapid successive saves; on validate failure, previous Snapshot is retained and WARN is logged — bot keeps running |
| T-02-04 | Tampering | config symlink swap | mitigate | Resolve config path once at startup with filepath.Abs; log resolved path on every reload; fsnotify watches the parent directory so symlink swaps produce a Create/Rename event that triggers reload with the new content |
| T-02-05 | Information Disclosure | config YAML contents | accept | Config contains group JID and owner JIDs — low-value PII; file permissions are operator responsibility; no credentials or tokens in Phase 1 config |
| T-02-SC | Tampering | go module installs | mitigate | All packages pinned in go.mod with go.sum checksum verification; pre-audited in RESEARCH.md Package Legitimacy Audit |
</threat_model>
