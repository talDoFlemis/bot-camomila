    - Records startTime = time.Now() immediately
    - Returns &Adapter{cfg: cfg, startTime: time.Now()}
    - Does NOT open DB or connect yet — those happen in Start()

    Implement Start(ctx context.Context) error:
    1. Get DB path from cfg.Get().DB.Path.
    2. Open DB: dsn := "file:" + dbPath + "?_foreign_keys=on"; db, err := sql.Open("sqlite", dsn). Wrap errors with fmt.Errorf("open sqlite: %w", err). Store db on a.db.
    3. PRAGMA integrity_check: var result string; db.QueryRowContext(ctx, "PRAGMA integrity_check").Scan(&result). If err != nil or result != "ok": db.Close(); return fmt.Errorf("SQLite integrity check failed: result=%q err=%v", result, err).
    4. Create sqlstore container: sqlstore.NewWithDB(db, "sqlite3", newWALogger("sqlstore")). The dialect string is "sqlite3" (the alias, NOT "sqlite").
    5. Call container.Upgrade(ctx). Wrap errors.
    6. Call container.GetFirstDevice(ctx). Wrap errors.
    7. Create whatsmeow client: whatsmeow.NewClient(device, newWALogger("whatsmeow")).
    8. Store cancel func from ctx: extract the cancel from the context using a context.WithCancel wrapper. Store it as a.cancel.
    9. Register event handler: a.client.AddEventHandler(a.onEvent).
    10. QR or Connect check: if a.client.Store.ID == nil (first run, no stored session), call a.client.GetQRChannel(ctx) then launch goroutine to range over the channel: for each item where item.Event == "code", print to stdout: fmt.Println("Scan this QR code or paste into a QR generator:") followed by fmt.Println(item.Code).
    11. Call a.client.Connect(). Return the error.

    Implement Disconnect():
    - If a.client != nil, call a.client.Disconnect(). This is ONLY called from outside the event handler (by app.go on ctx.Done()).
    - If a.db != nil, call a.db.Close().

Then use the derived ctx for all subsequent calls (db open, integrity check, sqlstore upgrade). When a.cancel() is called from the event handler (on LoggedOut/StreamReplaced), it cancels this derived context and propagates up.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WhatsApp network → event handler | Inbound events from the WhatsApp protocol are untrusted input |
| SQLite file → session store | Corrupt or tampered SQLite file could inject bad session state |
| QR code display → terminal | QR string printed to stdout; must not embed control characters from malicious server response |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | onEvent group JID gate | mitigate | Gate 1 in handleMessage checks evt.Info.Chat.String() == snap.Scope.GroupJID; snap is loaded atomically; mismatched JID drops immediately with structured log |
| T-03-02 | Denial of Service | HistorySync replay flood | mitigate | Gate 0 (timestamp gate, D-07) drops all events with Timestamp.Before(startTime); startTime recorded in New() before any Connect call |
| T-03-03 | Denial of Service | self-reply loop | mitigate | Gate 2 (IsFromMe) drops bot's own messages; checked after group gate, before text extraction |
| T-03-04 | Denial of Service | SQLite corruption on startup | mitigate | PRAGMA integrity_check runs before sqlstore.NewWithDB; result != "ok" causes Start() to return error and bot refuses to start (SESSION-03) |
| T-03-05 | Tampering | Forged owner JID in inbound messages | mitigate | evt.Info.Sender.ToNonAD().String() is used for logging; owner JID comparison (Phase 3) must also use ToNonAD() to strip device suffix — established here as the convention |
| T-03-06 | Denial of Service | Disconnect() deadlock from event handler | mitigate | LoggedOut and StreamReplaced call a.cancel() only; Disconnect() is never called from within the event handler goroutine (called by app.go after ctx.Done()) |
| T-03-07 | Information Disclosure | SQLite session DB file | mitigate | DB opened with file: URI; permissions controlled by host OS; .dockerignore will exclude *.sqlite* in Phase 4 |
| T-03-SC | Tampering | go module installs | mitigate | All packages pinned; go.sum checksum verification; pre-audited in RESEARCH.md |
</threat_model>
