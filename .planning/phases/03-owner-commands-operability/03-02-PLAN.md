       - Compute senderJID := evt.Info.Sender.ToNonAD().String()

         slog.Debug("owner command denied", "event", "dispatch", "reason", "owner_command_denied", "sender_jid", senderJID, "cmd", cmd)

       - slog.Info("owner command executed", "event", "dispatch", "reason", "owner_command", "sender_jid", senderJID, "cmd", cmd, "ack", ackText)
       - go a.sendCommandAck(evt, ackText)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WhatsApp group message → handleMessage() | Untrusted sender JIDs and message text arrive from the wire |
| handleMessage() → handleOwnerCommand() | Only reached if text matches !pause or !resume after trim+lowercase |
| handleOwnerCommand() → ownercommands.Handle() | Only reached after auth check; unauthorized callers return before this line |
| GetGroupInfo() network call | Outbound call to WhatsApp servers; fail-closed on error |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Spoofing | owner JID check in handleOwnerCommand() | mitigate | Always use evt.Info.Sender.ToNonAD().String() for senderJID; compare against listener.OwnerJIDs stored JIDs. Prevents AD-suffix bypass (RESEARCH.md Pitfall 2). |
| T-03-02-02 | Elevation of Privilege | allow_admin_commands GetGroupInfo path | mitigate | Admin check is fail-closed: GetGroupInfo error → unauthorized. Compare both p.JID.ToNonAD() and p.LID.ToNonAD() to prevent LID-based bypass (RESEARCH.md Open Question 1). |
| T-03-02-03 | Denial of Service | !pause command from authorized sender | accept | Authorized owner can pause the bot; that is the intended capability. Resume is equally accessible to the same owner. |
| T-03-02-04 | Tampering | kill switch state during hot-reload | mitigate | ks *killswitch.Switch passed once to New(), never re-created on config reload. config.Watcher only calls cfgStore.Set(); it has no reference to ks. |
| T-03-02-05 | Repudiation | command execution | mitigate | slog.Info with reason:"owner_command", sender_jid, cmd fields provides an audit trail. Denied attempts logged at Debug with reason:"owner_command_denied". |
| T-03-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages; all deps already in go.mod. Package Legitimacy Audit in RESEARCH.md confirms zero new deps. |
</threat_model>
