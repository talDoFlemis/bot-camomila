<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SIGTERM / SIGINT → shutdown handler | OS signals trigger cleanup; must complete within 10s |
| config.Load() → atomic swap | Single writer (watcher goroutine) updates snapshot; all readers (event handler) load atomically |
| adapter.Start() → WhatsApp connection | Network connection to WhatsApp servers; any error must propagate cleanly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | app.Run() shutdown ordering | mitigate | adapter.Disconnect() called after ctx.Done() (never from event handler); db.Close() called inside Disconnect(); 10s graceful period enforced by OS-level SIGKILL if shutdown hangs |
| T-04-02 | Tampering | config.Load() initial load failure | mitigate | app.Run() returns error immediately if initial config load fails; cmd/bot/main.go logs ERROR and exits non-zero; bot never starts with invalid config |
| T-04-03 | Denial of Service | watcher goroutine panic | mitigate | Watcher.Run() error is logged at ERROR level; bot continues running with last valid snapshot; watcher goroutine failure does not crash bot |
| T-04-04 | Elevation of Privilege | forged LoggedOut event causing premature exit | accept | LoggedOut triggers a.cancel() which initiates graceful shutdown; this is the correct response to a genuine logout event; no authentication of the event source beyond whatsmeow's Signal encryption |
| T-04-05 | Information Disclosure | config path exposed via --config flag | accept | Config path is operator-controlled and visible in process list; no credentials in path; low-value target |
| T-04-SC | Tampering | go module installs | mitigate | Pinned go.mod + go.sum checksum verification; all packages pre-audited |
</threat_model>
