## Overview

bot-camomila is a WhatsApp de-escalation bot for a single group chat. It connects via the whatsmeow multi-device protocol, fuzzy-matches keywords in messages using Levenshtein distance, and replies with a randomly-picked calming response — threaded to the triggering message. Named for chamomile tea.

The architecture follows a **hexagonal (ports & adapters)** pattern: the WhatsApp adapter is the only package that imports whatsmeow. Everything else is pure Go, testable without a WhatsApp connection.

## Data Flow

### Message Processing Pipeline (Phase 1 — gates only)

1. **WhatsApp** delivers raw event to `whatsmeow.Client`
2. **`onEvent()`** type-switches: dispatches `*events.Message` → `handleMessage()`; handles lifecycle events (Connected, Disconnected, LoggedOut, StreamReplaced, PairSuccess)
3. **Gate 0 — HistorySync filter:** drops messages with timestamp before `a.startTime`
4. **Gate 1 — Group JID scope:** drops messages not from the configured group JID
5. **Gate 2 — Self-message filter:** drops `IsFromMe == true` (self-reply loop prevention)
6. **Gate 3 — Text-only filter:** extracts text via `extractText()` (conversation or extended text); drops non-text messages
7. **Terminal action (Phase 1):** logs structured message receipt. Phase 2 will add matcher pipeline dispatch and reply here.

### Config Hot-Reload Flow

1. **fsnotify** watches the parent directory of `config.yaml` (survives atomic-rename saves)
2. On qualifying event (Write/Create/Rename for the config file basename) → 200ms debounce timer reset
3. After debounce → `reload()` calls `Load()` → if validation passes, `Store.Swap()` atomically replaces the snapshot
4. On failure → WARN log, previous snapshot retained
5. Fallback: if fsnotify is unavailable or errors, 30s mtime-polling loop takes over

### Startup Sequence

1. `main.go`: setup logging → parse `--config` flag / `BOT_CONFIG` env → record `startTime` → signal context
2. `app.Run()`: load initial config → create `Store` → start `Watcher` goroutine → create `Adapter` → `adapter.Start()`
3. `adapter.Start()`: open SQLite → `PRAGMA integrity_check` → create `sqlstore` container → upgrade schema → get device → create client → register event handler → QR or resume → `Connect()`
4. Block on `<-ctx.Done()` → `adapter.Disconnect()` → exit

### Shutdown Sequence

1. `SIGINT`/`SIGTERM` → context cancelled
2. `app.Run()` unblocks from `<-ctx.Done()`
3. `adapter.Disconnect()` → `client.Disconnect()` → `db.Close()`
4. Process exits cleanly
